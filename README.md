# Riku-Ryu Corpus
**Corpus-Based Vocabulary — QuestQueries #031**

> 「規則を教えてから例文」ではなく「例文の海から規則が浮かび上がる」

---

## コンセプト

英語学習における根本的な問いへのアプローチ：

**「なぜ listen と hear はどちらも『聞く』なのに、使い分けが難しいのか？」**

辞書的な定義の説明では解決しない。コーパス（用例の集積）を大量に見ることで、体が先に覚える。それがこのアプリの哲学です。

さらに、**専門分野に特化**することで、汎用英語教材では絶対にカバーされない領域をカバーします。歴史学者が使う語彙と、物理学者が使う語彙は違う。そのジャンルで必要な単語ペアを、AIが選び出します。

---

## 機能

### ① 職業選択
8つの専門職から選ぶ（または自由入力予定）。選択した専門分野に合わせて、AIが単語ペアを提案。

### ② 単語ペア提案
「同じ日本語に訳されるが、英語では運用が違う」ペアを8組提示。汎用教材では出てこない、専門文脈に特化した選定。

### ③ 例文表示（先行ロード）
選んだペアについて、各単語6文（教科書3＋論文3）を左右並列で表示。解説なし。文脈の中で違いを感じとる。

タブ切り替えで「すべて / 📖教科書 / 🎓論文」にフィルタリング可能。

### ④ 副詞コロケーション（後追いロード）
例文表示後、バックグラウンドで取得。「この単語とよく一緒に使われる副詞」を専門文脈つきで5個表示。

### ⑤ 違いのまとめ（後追いロード）
副詞と並列取得。例文を読んだ後に「💡 結局どう違うの？」として日本語で直感的な説明を表示。文法解説ではなく、感覚的なまとめ。

---

## ローディング設計（体感速度の工夫）

```
ユーザーがペアを選択
  ↓
[~10秒] 例文12本を取得・表示  ← ここで読み始められる
  ↓ 並列バックグラウンド
[+~10秒] 副詞コロケーション + まとめ を後追い表示
```

30秒待って全部出るより、10秒で読み始めて追いかけて揃う体験を優先。

## APIコール構成

| # | タイミング | 内容 | max_tokens |
|---|-----------|------|-----------|
| ① | 職業選択時 | 単語ペア8組を生成 | 1000 |
| ② | ペア選択時（先行） | 例文12本を生成 | 2000 |
| ③ | ②完了後（並列） | 副詞コロケーション | 1000 |
| ④ | ②完了後（並列） | 違いのまとめ（日本語） | 600 |

## セットアップ

```
1. AnthropicのAPIキーを取得
2. ファイルをClaudeのArtifactまたはReact環境で開く
3. APIキーを環境変数またはヘッダーとして設定
```

> ⚠️ ブラウザから直接APIを叩く構成のため、APIキーの取り扱いに注意。
> GitHub Pagesで公開する場合は、キーを入力フォームで受け取る形にすること。
